Emerging & Lightweight Tools
Tool Type Status What It Does Best For
cc-sdd CLI Open Source Kiro-style SDD workflow (npx cc-sdd@latest) for 8 different agents/IDEs. Requirements → Design → Tasks with steering files. Want Kiro's workflow without switching IDEs. Cursor, Claude Code, Copilot, Gemini CLI.
Autospec CLI Open Source Spec Kit-inspired CLI for Claude Code with YAML validations, retries, notifications, streamlined flows. Claude Code users wanting lightweight spec workflow.
OpenSpec Framework Open Source Lightweight spec format for AI coding agents by Fission AI. Emphasis on simplicity and portability. Teams wanting minimal spec overhead with maximum agent compatibility.
PromptX Framework Open Source Prompt engineering patterns structured as reusable specifications. Focuses on prompt architecture. Teams focused on prompt/context engineering discipline.
SPARC + Claude-Flow Framework Open Source Specification, Pseudocode, Architecture, Refinement, Completion framework. Pairs with Claude-Flow for orchestration. Claude Code power users wanting structured multi-phase workflow.

Alignment with your methodology: Your Phase 0 → 0.5 → 1 workflow maps closely to the Specify → Plan → Tasks → Implement pattern these tools formalize. Given your Cursor-based workflow, GitHub Spec Kit and cc-sdd are the most natural fits — agent-agnostic, lightweight, no IDE switch required. BMAD is worth evaluating if multi-agent orchestration appeals. Tessl's Spec Registry is independently useful for preventing library hallucinations regardless of which SDD tool you adopt.
